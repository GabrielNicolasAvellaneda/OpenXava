@SearchKey

Now it's possible to mark a property as search key. The searching properties will be used by the user for searching in the user interface of the references.
That is, you now can write a code like this one for an Invoice for example:
    @Id @GeneratedValue(generator="system-uuid") @Hidden 
    @GenericGenerator(name="system-uuid", strategy = "uuid")
    private String oid;

    @SearchKey
    @Column(length=4) @DefaultValueCalculator(CurrentYearCalculator.class)
    private int year;
    
    @SearchKey
    @Column(length=6)
    private int number;
And when the user will be search Invoice from another entity, the user will can type year and number in order to search it.
