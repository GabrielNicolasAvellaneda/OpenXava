El equipo de OpenXava es:
  • Gestión cuatrocientos(Ana Andrés, Daniel Gimeno, Ivan Martín, Javier Paniza, Lars Lathan, Luis Miguel Hernández, Miguel Angel Embuena): Todo el código de OpenXava excepto lo que se cita abajo.
  • Bernabé Torres Ayala: Editor radioButton para valores-posibles.
  • Carlos Gascó, Carlos Díaz: Ayuda en las traducciones a Inglés de v1.0.
  • Christoph Ruckert: El atributo en-cada-peticion para las acciones en controlladores.xml, el estereotipo ETIQUETA_IMAGEN y buttonsForNoImageActions para xava.properties.
  • Eduardo Escrihuela: Calculador CurrentTimestampCalculator.
  • Hans-Joerg Bauer: Corregir errores en el código y la revisión gramatical de la guía de referencia en inglés.
  • Janesh Kodikara: Estereotipos EMAIL, TELEFONO, WEBURL, IP, ISBN, TARJETA_CREDITO,  LISTA_EMAIL, el calculador DaysDelayCalculator, el conversor StringTimeConverter y la utilidad Emails.
  • Javier Paniza: Versión inicial (un generador de aplicaciones swing desde XML), generador de código para EJB1.1/2.0, interfaz de usuario Web, guía de referencia en inglés y español para v1.1.
  • José Enrique Romero Flores: Estereotipo HTML_TEXT/TEXTO_HTML y la integración con el portal Stringbeans.
  • José Luis Santiago: Versión para Liferay Portal y otras aportaciones al código.
  • José Puente Ferrandis: Plugin de Eclipse (Navigator) con asistentes para crear proyectos OX y editores Eclipse para archivos OX.
  • Lars Lathan: Traducción a alemán.
  • Mª Carmen Gimeno Alabau: Gestor de persistencia para Hibernate, generador de código POJO, generador de mapeos de Hibernate y el proyecto demo ProductSale.
  • Matías Gagliano: Vínculo para mostrar/ocultar la parte del filtro en la lista y las colecciones.
  • Newitech Sp. z o.o. (Radoslaw Ostrzycki, Lukasz Milobedzki): Jakarta Commons Loggin como sistema de logging, atributo 'escala' en las propiedades, generación de portlets con UTF-8, mejorar la generación del mapeo hibernate, traducción al polaco y más.
  • Sami AlSayyed: Protección contra XSS.
  • Sergio Mondino: Plugin de Eclipse (Importer) para crear componentes OX desde las tablas de la base de datos.
  • Trifon Trifonov: final staticspara propiedades en las interfaces en la generación de código, el atributo longitud-visual en vista-propiedad, el soporte de búlgaro en la generación de portlets y mostrar las etiquetas de las propiedades al añadir columnas a la lista.
