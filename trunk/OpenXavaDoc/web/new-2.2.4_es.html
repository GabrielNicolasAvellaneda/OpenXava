Claves de búsqueda

Ahora es posible marcar una propiedad como clave búsqueda. Las propiedades de búsqueda se usarán por el usuario para buscar en la interfaz de usuario por referencias.
Es decir, podemos escribir un código como esto para una Factura por ejemplo:
	<propiedad nombre="oid" tipo="String" clave="true" oculta="true">
            <calculador-valor-defecto clase="org.openxava.calculators.UUIDCalculator" on-create="true"/>
        </propiedad>

        <propiedad nombre="año" tipo="int" clave-busqueda="true" longitud="4">
            <calculador-valor-defecto clase="org.openxava.calculators.CurrentYearCalculator"/>    
        </propiedad>
        
        <propiedad nombre="numero" tipo="int" clave-busqueda="true" longitud="6"/>
Y cuando el usuario busque una Factura desde otra entidad, el usuario podrá teclear el año y el número para poder buscarla.

Acceder a la información del usuario del portal

Hay una API simple para acceder a la información del usuario actual desde una aplicación OpenXava:
UserInfo userInfo = Users.getCurrentUserInfo();
userInfo.getGivenName();
userInfo.getEmail();
userInfo.getFamilyName();
Si el módulo no se está ejecutando dentro del portal el objeto userInfo estará vacío, pero no será nulo.
