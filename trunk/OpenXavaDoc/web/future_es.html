Versión 3

La próxima versión de OpenXava permitirá definir componentes usando POJOs (Entidades EJB3) + anotaciones, como alternativa al XML. La ventajas son:
  • Se podrá coger una aplicación desarrollada con EJB3 y convertirla en una aplicación OpenXava en cuestión de minutos.
  • Tendremos la posibilidad de usar herramientas estándar para POJO/EJB3 para desarrollar aplicaciones OpenXava. Herramientas como dibujadores UML, Dali para el mapeo, Hibernate tools para importar esquemas a EJB3 POJOs, etc.
  • No necesitaremos usar generación de código.
  • Podremos desarrollar usando Java.

Otras áreas interesante para desarrollar

Estas características todavía no están planificadas, pero son muy interesantes, si quieres implementar alguna estás invitado.
  • Plugins para Eclipse: Ya tenemos un plugin de Eclipse para OpenXava pero puede ser mejorado. Por ejemplo, la posibilidad de crear un proyecto nuevo de OpenXava desde el Eclipse es algo fácil de implementar y puede ser una herramienta útil.
  • Anotaciones: Mejorar OX3 con anotaciones como @Property, @Reference, @Collection, @Calculator, @Converter, @DefaultValue. Estas anotaciones pueden usar APT para manipular el bytecode o algo parecido.
  • Aplicación final a estilo Wiki: La posibilidad para el usuario final de añadir propiedades, referencias y colecciones él mismo.
  • Generación automática de servicios web: Esto es fácil de implementar y útil.
  • Una versión FLEX
  • Una herramienta desarrollada con OpenXava para desarrollar aplicaciones OpenXava.
  • WSRP
  • Integración con Workflow
  • MDA/EMF
  • Una colección de componentes de negocio listos para usar desarrollados con OpenXava: Esto pudiera ser un proyecto independiente.
