It's possible to specify that certain action in a collection will not be shown

You can write:
<view>
  <collection-view collection="myCollection">
     <remove-selected-action action=""/>
  </collection-view>        
  ...
</view>
In this case the action to remove selected elements in this collection will not be shown in the User Interface. Only it's need to specify an empty string as value for the action.
This apply to all actions in <collection-view/> that is to: <edit-action/>, <view-action/>, <new-action/>, <save-action/>, <hide-detail-action/>, <remove-action/> and <remove-selected-action/>.

Display size for a property in a view

Now you can use display-size attribute for <property-view/>, in this way you can restrict the display size of a property for a specific view. That is, you can write:

<entity>
		
...
  <property name="description" type="String
    size="50" required="true"/>
	
</entity>
	 
	
<view>
  <property-view property="description" display-size="25"/>
</view>

In this case, although 'description' has a size of 50 the size of the editor for 'description' will be only 25, even though the user can enter until 50 characters.
