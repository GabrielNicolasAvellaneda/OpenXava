OpenXava has no security or user management system
But you can use the security of your underlaying Java Portal. You should consult your portal documentation about security. Anyways here there are some tips for JetSpeed2

JetSpeed security and OpenXava

When OpenXava deploy portlets also generates the propietary JetSpeed2. These files are in openxava.war/WEB-INF/pages inside a folder named as the proyect. For example, is your application is MySchool this files are in openxava.war/WEB-INF/pages/MySchool
These files are configured to grant all access to the application to user of group 'MySchool', hence if you wish grant or deny access to your application you must to create a new group (with JetSpeed Administrative Portlets) named 'MySchool' and assign to it all the users of your election.
If you need a more granulated security, you can edit these files and to modify them. See documentation about this in JetSpeed 2 Site. Beware, because the jetspeed2 files are overwrited by OpenXava when you deploy portlets again. Hence you need to configure your ant task in this way: 
	<target name="deployPortlets">
		<ant antfile="../OpenXava/build.xml" target="deployPortlets">
			<property name="generate.jetspeed2.files" value="false"/>
		</ant>		
	</target>
