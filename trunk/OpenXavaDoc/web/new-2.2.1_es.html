Users.getCurrent() puede devolver el correo electrónico del usuario dentro del portal

El id de usuario dentro de algunos portals (como Liferay o WebSphere Portal bajo ciertas circunstancias) puede ser a id interno con poco significado (como liferay.com.1001). Sin embargo podemos configurar nuestra aplicación OpenXava para devuelva la cuenta de correo electrónico del usuario (en vez del id) con el método Users.getCurrent().
Sólo hemos de editar nuestro archivo xava.properties y poner:
emailAsUserNameInPortal=true

Es posible especificar que cierta acción en una colección no se muestre

Podemos escribir:
<vista>
  <vista-coleccion coleccion="miColeccion">
     <accion-quitar-seleccionados accion=""/>
  </vista-coleccion>        
  ...
</vista>
En este caso la acción para borrar elementos seleccionado en esta colección no será mostrada en la interfaz de usuario. Sólo es necesario especificar una cadena vacía como valor para la acción.
Esto aplica a todas las acciones en <vista-coleccion/> es decir a <accion-editar/>, <accion-ver/>, <accion-nuevo/>, <accion-grabar/>, <accion-ocultar-detalle/>, <accion-quitar/> y <accion-quitar-seleccionados/>.

Longitud de visualización para una propiedad en una vista

Ahora podemos usar el atributo longitud-visual para <vista-propiedad/>, de esta forma podemos restringir el tamaño de visualización de una propiedad para una vista específica. Es decir, podemos escribir:
<entidad>	
...
  <propiedad nombre="descripcion" tipo="String"
    longitud="50" requerido="true"/>
	
</entidad> 
	
<vista>
  <vista-propiedad propiedad="descripcion" longitud-visual="25"/>
</view>

En este caso, aunque 'descripcion' tiene un tamaño de 50 el tamaño del editor para 'descripcion' será de solo 25, aun así el usuario puede introducir hasta 50 caracteres.
