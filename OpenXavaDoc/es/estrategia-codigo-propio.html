¿Cómo se puede insertar nuestra funcionalidad personalizada en un componente OpenXava?

Las piezas azules son objetos que se pueden insertar en nuestros componentes, para que tengan nuestra propia funcionalidad. Usamos el patrón clásico de diseño llamado Estrategia (Strategy o Policy).
Estos son los objetos estratégicos que podemos insertar:
  • Calculadores (ICalculator): Implementan lógica de negocio. Usados para propiedades calculadas, para calcular valores por defecto o definir métodos.
  • Validadores (IValidator): Para validar los datos introducidos por el usuarios. Podemos definirlos a nivel de propiedad o a nivel de modelo.
  • Conversores (IConverter): Permiten definir la conversión del tipo Java al tipo de la base de datos. Usado en el mapeo. Muy util cuando usamos bases de datos legadas.
  • Filtros (IFilter): Permiten filtrar con nuestro código los valores enviado al gestor de datos tabulares para ejecutar una consulta. Imprescindible para hacer aplicaciones reales.
Además de en los componentes hay otras clases que nos permiten definir el comportamiento de una aplicación OpenXava:
  • Acciones (IAction): Son los objetos que reaccionan a las peticiones del usuario. Se definen en controladores.xml.
  • Formateadores (IFormatter): Con un formateador podemos especificar como convertir de un formulario html (String) al tipo Java y viceversa. Así podemos definir el formato de visualización de los datos y su transformación a Java.
  • Generador (IPropertyCodeGenerator): Podemos definir nuestro propio generador de código para ciertas propiedades.
Todos estos objetos de estrategía se pueden configurar desde el XML dando valores a sus propiedades. De esa forma podemos hacer objetos estrategía configurables que podemos reutilizar varias veces. Así tenemos más XML y menos Java.
De esta forma se tiene suficiente funcionalidad para adaptarse a practicamente cualquier necesidad de una aplicación de gestión real. Pero además, existe la posibilidad de escribir a mano de forma íntegra el EJB con la lógica de negocio, o si lo deseamos la interfaz gráfica usando directamente JSP. Aunque esta virtud existe  para aprovechar código ya hecho y no suele ser necesario cuando se escribe código nuevo.
