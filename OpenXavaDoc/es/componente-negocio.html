Los componentes de negocio son la únidad básica sobre la que se construye una aplicación OpenXava.
Un componente de negocio (o un componente OpenXava, si se prefiere); aglutina toda la información relativa a un concepto de negocio.
Supongamos el concepto de negocio de Factura, pues bien; todas la información acerca de una Factura se define en Factura.xml. Ahí definimos las validaciones, los datos que se almacena, la lógica de negocio, el interfaz gráfico, los datos tabulares, el mapeo a la base de datos, etc. etc.
J2EE propopone una división con desarrolladores especializados técnicamente:un diseñador gráfico, uno programador para el comportamiento de la aplicaciones, programadores para la lógica de negocio, desplegadores (que configuran la aplicación), etc. Por lo contratario, en OpenXava, se propone una división en base a la funcionalidad, y se deja a OpenXava lidiar con los aspectos más técnicos (interfaz gráfico, acceso a datos, etc). Por la práctica hemos visto más eficiente este enfoque, sobre todos cuando se trabaja en equipo usando CVS.
