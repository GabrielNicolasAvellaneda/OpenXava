Tutorial OpenXava para la distribución 'core'

La distribución 'core' tiene el marco de trabajo OpenXava completo con pruebas y ejemplos. Está pensada para ser usada por usuarios avanzados de OpenXava bien para actualizar su versión de OpenXava a la última, o simplemente porque prefieren configurar su propio portal o servidor de aplicaciones.
Si eres nuevo en Java es mucho mejor que uses la distribución openxava-x.x.x-jetspeed2 que viene con Tomcat + JetSpeed + Workspace de Eclipse todo configurado y listo para usar, más un portal local de OpenXava con documentación, guías de referencia, bases de datos, etc.
Aunque si conoces bien Java, Ant y Tomcat puedes seguir este tutorial y saborear OpenXava.
Estas instrucciones suponen que tienes:
  • Java JDK 5 o JDK 1.4.2.
  • Tomcat 5.5 o 5.0 instalado y listo para funcionar.
  • Ant 1.6.x.
Esto es lo que vamos a hacer para empezar a xavear:
  • Configurar bases de datos en Tomcat.
  • Ejecutar aplicación ejemplo MiEscuela.
  • Modificar aplicación ejemplo MiEscuela.
  • Ayuda

Configurar bases de datos en Tomcat

El primer paso es crear y configurar las bases de datos para ejecutar la aplicación de demo:
  • Crear una carpeta llamada data dentro del Tomcat.
  • Copiar desde OpenXava/data los archivos openxava-db.script y openxava-db.properties a la carpeta data del Tomcat.
  • Copiar desde MySchool/data los archivos my-school-db.script y my-school-db.properties a la carpeta data del Tomcat.
Edita el archivo conf/context.xml de tu Tomcat para declarar las fuentes de datos, y añade las dos siguiente:

<Context>
    ...
    <Resource name="jdbc/OpenXavaDS" auth="Container" type="javax.sql.DataSource"
       maxActive="20" maxIdle="5" maxWait="10000"
       username="sa" password="" driverClassName="org.hsqldb.jdbcDriver"
       url="jdbc:hsqldb:file:../data/openxava-db"/>

    <Resource name="jdbc/MiEscuelaDS" auth="Container" type="javax.sql.DataSource"
       maxActive="20" maxIdle="5" maxWait="10000"
       username="sa" password="" driverClassName="org.hsqldb.jdbcDriver"
       url="jdbc:hsqldb:file:../data/my-school-db"/>

</Context>

Ahora falta copiar el archivo OpenXavaTest/lib/hsqldb.jar a la carpeta common/lib del Tomcat.
La base de datos ya está lista.

Ejecutar aplicación ejemplo MiEscuela

Primero, necesitas editar el archivo openxava.properties (en la carpeta raiz de openxava) y poner en la propiedad tomcat.dir la ruta de tu Tomcat.
La distribución de OpenXava viene con una aplicación de ejemplo llamada MiEscuela, para ejecutar sigue los siguientes pasos, usando Ant desde la línea de ordenes:
  • cd MiEscuela
  • ant
  • Arranca tu Tomcat.
  • Ir a http://localhost:8080/MiEscuela/xava/module.jsp?application=MiEscuela&module=Profesores
  • Y a jugar...

Modificar aplicación ejemplo MiEscuela

Añadir otro componente y su mantenimiento
Copiamos el archivo components/Profesor.xml y lo pegamos dandole el nombre de Alumno.xml.
Cambiamos Alumno.xml hasta que quede de la siguiente forma:

<?xml version="1.0" encoding="ISO-8859-1"?>

<!DOCTYPE componente SYSTEM "dtds/componente.dtd">

<componente nombre="Alumno">

    <entidad>
        <propiedad nombre="numero" tipo="int" clave="true"
            longitud="2" requerido="true"/>
        <propiedad nombre="nombre" tipo="String"
            longitud="40" requerido="true"/>             
    </entidad>
                       
    <mapeo-entidad tabla="MYSCHOOL@separator@PUPIL">
        <mapeo-propiedad propiedad-modelo="numero" columna-tabla="NUMBER"/>
        <mapeo-propiedad propiedad-modelo="nombre" columna-tabla="NAME"/>
    </mapeo-entidad>
               
</componente>

Lo marcado en azul es lo que hay que cambiar.
También tenemos que editar el archivo xava/aplicacion.xml, y añadir un nuevo módulo:

<?xml version = "1.0" encoding = "ISO-8859-1"?>

<!DOCTYPE aplicacion SYSTEM "dtds/aplicacion.dtd">

<aplicacion nombre="MiEscuela">
   
    <modulo nombre="Profesores">
        <modelo nombre="Profesor"/>
        <controlador nombre="Typical"/>
    </modulo>     
   
    <modulo nombre="Alumnos">
        <modelo nombre="Alumno"/>
        <controlador nombre="Typical"/>
    </modulo>     
                   
</aplicacion>

Lo marcada en azul es lo que hay que añadir.
Probemos nuestro nuevo módulo:
  • ant
  • Ve a http://localhost:8080/MiEscuela/xava/module.jsp?application=MiEscuela&module=Alumnos
  • Y a jugar...
Que los alumnos y los profesores se conozcan
Es sencillo hacer que un alumno tenga una referencia a un profesor, modificamos el archivo Alumno.xml para que quede así:

<?xml version="1.0" encoding="ISO-8859-1"?>

<!DOCTYPE componente SYSTEM "dtds/componente.dtd">

<componente nombre="Alumno">

    <entidad>
        <propiedad nombre="numero" tipo="int" clave="true"
            longitud="2" requerido="true"/>
        <propiedad nombre="nombre" tipo="String"
            longitud="40" requerido="true"/>             
        <referencia nombre="profesor"/>   
    </entidad>
                       
    <mapeo-entidad tabla="MYSCHOOL@separator@PUPIL">
        <mapeo-propiedad propiedad-modelo="numero" columna-tabla="NUMBER"/>
        <mapeo-propiedad propiedad-modelo="nombre" columna-tabla="NAME"/>
        <mapeo-referencia referencia-modelo="profesor">
            <detalle-mapeo-referencia columna-tabla="TEACHER" propiedad-modelo-referenciado="codigo"/>
        </mapeo-referencia>
    </mapeo-entidad>
               
</componente>

Y ahora lo probamos:
  • ant
  • Ve a http://localhost:8080/MiEscuela/xava/module.jsp?application=MiEscuela&module=Alumnos
Como se ve, harto sencillo.
Si queremos que la referencia se visualice en formato combo hemos de cambiar Alumno.xml para que quede así:

<?xml version="1.0" encoding="ISO-8859-1"?>

<!DOCTYPE componente SYSTEM "dtds/componente.dtd">

<componente nombre="Alumno">

    <entidad>
        <propiedad nombre="numero" tipo="int" clave="true"
            longitud="2" requerido="true"/>
        <propiedad nombre="nombre" tipo="String"
            longitud="40" requerido="true"/>             
        <referencia nombre="profesor"/>   
    </entidad>
   
    <vista>
        <vista-referencia referencia="profesor">
            <lista-descripciones/>
        </vista-referencia>
    </vista>
                       
    <mapeo-entidad tabla="MYSCHOOL@separator@PUPIL">
        <mapeo-propiedad propiedad-modelo="numero" columna-tabla="NUMBER"/>
        <mapeo-propiedad propiedad-modelo="nombre" columna-tabla="NAME"/>
        <mapeo-referencia referencia-modelo="profesor">
            <detalle-mapeo-referencia columna-tabla="TEACHER" propiedad-modelo-referenciado="codigo"/>
        </mapeo-referencia>
    </mapeo-entidad>
               
</componente>

Y desplegar:
  • ant
  • Ve a http://localhost:8080/MiEscuela/xava/module.jsp?application=MiEscuela&module=Alumnos
Y ahora vamos a poner una colección de alumnos en el profesor. Tocamos Profesor.xml y lo dejamos así:

<?xml version="1.0" encoding="ISO-8859-1"?>

<!DOCTYPE componente SYSTEM "dtds/componente.dtd">

<componente nombre="Profesor">

    <entidad>
        <propiedad nombre="codigo" tipo="String" clave="true"
            longitud="5" requerido="true"/>
        <propiedad nombre="nombre" tipo="String"
            longitud="40" requerido="true"/>             
        <coleccion nombre="alumnos">
            <referencia modelo="Alumno"/>
        </coleccion>
    </entidad>
                       
    <mapeo-entidad tabla="MYSCHOOL@separator@TEACHER">
        <mapeo-propiedad propiedad-modelo="codigo" columna-tabla="ID"/>
        <mapeo-propiedad propiedad-modelo="nombre" columna-tabla="NAME"/>
    </mapeo-entidad>
               
</componente>

Volvemos a desplegar:
  • ant
  • Ve a http://localhost:8080/MiEscuela/xava/module.jsp?application=MiEscuela&module=Profesores
Y eso es todo amigos.
Con este ejemplo se ve como cambios pequeños se efectuan con poco esfuerzo y el efecto se ve de forma instantanea.
