The OpenXava direction depends on its community.

Version 3

The next release of OpenXava will allow you define components using POJOs (EJB3 Entities) + annotations, as alternative to XML. The advantages are:
  • You can take your application developed using EJB3 and convert it in an OpenXava application in a few minutes.
  • You can use standard tools for POJO/EJB3 for developing OpenXava applications. Tools as UML tools, Dali for mapping, Hibernate tools for importing database schemas to EJB3 POJOs, etc.
  • You do not need code generation any more.
  • You can develop using Java.

Other interesting areas of development

These feature are not planed yet, but they are very interesting, if you want to implement some of them you  are invited to do it. 
  • Eclipse plugins: A Eclipse Plugin for OpenXava already exists but it can be improved. For example, the possibility of creating a new OpenXava project from Eclipse is easy to implements and it is an useful tool.
  • Annotations: Improving OX3 with annotation as @Property, @Reference, @Collection, @Calculator, @Converter, @DefaultValue. These annotations can use APT for manipulating bytecode or so.
  • End application at Wiki Style: The possibility for the final user to add properties, references and collection himself.
  • Automatic Web Services generation: Easy to implement and useful.
  • A FLEX version
  • A tool developed using OpenXava for developing OpenXava applications.
  • WSRP
  • Workflow integration
  • MDA/EMF
  • A library of ready to use business component developed using OpenXava: Maybe this can be an independent project.
