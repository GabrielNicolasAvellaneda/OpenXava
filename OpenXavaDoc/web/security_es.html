OpenXava no tiene seguridad o sistema de gestión de usuarios
Pero podemos usar la seguridad del portal Java subyacente. Deberías consultar la documentación de tu portal sobre seguridad. De todas formas aquí hay algunos trucos para usar con JetSpeed2.

Seguridad de JetSpeed y OpenXava

Cuando OpenXava despliega portlets también genera archivos propios de JetSpeed2. Estos archivos están en openxava.war/WEB-INF/pages dentro de una carpeta llamada como tu proyecto. Por ejemplo, si tu aplicación es MiEscuela estos archivos están en openxava.war/WEB-INF/pages/MiEscuela
Estos archivos están configurados para garantizar acceso total a la aplicación a los usuarios del grupo 'MiEscuela', por ende si quieres garantizar o denegar acceso a tu aplicación tienes que crear un nuevo grupo llamado 'MiEscuela' y asignar a él todos los usuarios que quieras. Esto lo puedes hacer con los 'Portlets administrativos Jetspeed'
Si necesitas una seguridad más granulada, puedes editar estos archivos y modificarlos. Mira lo documentación sobre ese tema en JetSpeed 2 Site. Ten cuidado, porque los archiovs de JetSpeed2 son sobreescritos por OpenXava cuando despliegas los portlets de nuevo. Por lo tanto necesitas configurar tu tarea ant de esta forma: 
<target name="desplegarPortlets">
	<ant antfile="../OpenXava/build.xml" target="deployPortlets">
		<property name="generate.jetspeed2.files" value="false"/>
	</ant>		
</target>

Restricción de datos por usuario

Si deseas restrigir los datos dependiendo del usuario actual, puedes usar técnicas de programación. Sobre esto puedes ver el código de org.openxava.filters.UserFilter. En este caso se usa request.getRemoteUser() para filtrar los datos.
