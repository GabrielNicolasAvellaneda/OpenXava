OpenXava no tiene seguridad o sistema de gestión de usuarios
Pero podemos usar la seguridad del portal Java subyacente. Deberías consultar la documentación de tu portal sobre seguridad. De todas formas aquí hay algunos trucos para usar con JetSpeed2.

Seguridad de JetSpeed y OpenXava

Cuando OpenXava despliega portlets también genera archivos propios de JetSpeed2. Estos archivos están en openxava.war/WEB-INF/pages dentro de una carpeta llamada como tu proyecto. Por ejemplo, si tu aplicación es Facturacion estos archivos están en openxava.war/WEB-INF/pages/Facturacion
. Todos los archivos (módulos definidos dentro de archivos PSML) en la carpeta raiz (en Facturacion) son accesibles a usuarios del grupo Facturacion, todos los archivos en la subcarpeta clientes (en Facturacion/clientes) son accesible a usuarios en el grupo Facturacion.clientes, todos los archivos en la subcarpeta informes de clientes (en Facturacion/clientes/informes) son accesibles a usuarios en el grupo Facturacion.clientes.informes y así por el estilo.
Solo necesitas usar los Portlets administrativos Jetspeed para crear los grupos de usuarios Facturas, Facturas.clientes y Facturas.clientes.informes y asignar tus usuarios a estos grupos a tu gusto.

Si necesitas una seguridad más granulada, puedes editar estos archivos y modificarlos. Mira lo documentación sobre ese tema en JetSpeed 2 Site. Ten cuidado, porque los archivos de JetSpeed2 son sobreescritos por OpenXava cuando despliegas los portlets de nuevo. Por lo tanto necesitas configurar tu tarea ant de esta forma: 

<target name="desplegarPortlets">
	<ant antfile="../OpenXava/build.xml" target="deployPortlets">
		<property name="generate.jetspeed2.files" value="false"/>
	</ant>		
</target>

Restricción de datos por usuario

Si deseas restrigir los datos dependiendo del usuario actual, puedes usar técnicas de programación. Sobre esto puedes ver el código de org.openxava.filters.UserFilter. En este caso se usa request.getRemoteUser() para filtrar los datos. Además, podemos usar la API de OpenXava Users.getCurrent() para obtener el nombre del usuario actual en cualquier parte de una aplicación OpenXava.
