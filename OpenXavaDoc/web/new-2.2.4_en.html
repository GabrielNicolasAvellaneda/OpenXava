Search keys

Now it's possible to mark a property as search key. The searching properties will be used by the user for searching in the user interface of the references.
That is, you now can write a code like this one for an Invoice for example:
        <property name="oid" type="String" key="true" hidden="true">
            <default-value-calculator class="org.openxava.calculators.UUIDCalculator" on-create="true"/>
        </property>

        <property name="year" type="int" search-key="true" size="4">
            <default-value-calculator class="org.openxava.calculators.CurrentYearCalculator"/>    
        </property>
        
        <property name="number" type="int" search-key="true" size="6"/>
And when the user will be search Invoice from another entity, the user will can type year and number in order to search it.

Accessing to the info of the portal user

There is a simple API for accessing to the info of the current user from an OpenXava application:
UserInfo userInfo = Users.getCurrentUserInfo();
userInfo.getGivenName();
userInfo.getEmail();
userInfo.getFamilyName();
If the module is not running inside the portal the userInfo object will be empty, but not null.
