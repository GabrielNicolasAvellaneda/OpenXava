These are some of the new features in OpenXava 2.2.3:
  • Optimistic concurrency support
  • Actions with no image can be displayed using buttons instead of links

Optimistic concurrency support

Concurrency is the ability of the application to allow several users to save data at same time without losing data. OpenXava uses an optimistic concurrency schema. Using optimistic concurrency the records are not locked allowing high concurrency without losing data integrety.

For example, if a user A read a record and then a user B read the same record, modify it and save the changes, when the user A try to save the record he receives an error, then he need to refresh the data and retry his modification.

For activating concurrency support for an OpenXava component you only need to declare a property using version=”true”, in this way:

<property name='version' type='int' version='true'/>

This property is for use of persistence engine (Hibernate or JPA), your application or your user must not use this property directly.

Only a single version property should be used per component. The following types are supported for version properties: int, Integer, short, Short, long, Long, Timestamp. The version properties are considered hidden.

Actions with no image can be displayed using buttons instead of links

OpenXava displays actions with no image associated to them using links, but now it's possible to configure your application in order that OX uses buttons instead. You only need to modify xava.properties and add:
buttonsForNoImageActions=true
