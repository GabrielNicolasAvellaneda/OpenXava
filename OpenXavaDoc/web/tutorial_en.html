Welcome to this short voyage to start to use OpenXava.
These instructions suposses  that openxava-1.1.zip is decompressed in OPENXAVA_HOME directory, eclipse in ECLIPSE_HOME and jboss in JBOSS_HOME.
Here are the steps to begin to use OpenXava:
  • Download base software needed.
  • Install and configure JDK.
  • Install and configure JBoss.
  • Install and configure Eclipse and OpenXava.
  • Execute example application MySchool.
  • Execute junit tests of MySchool application.
  • Modify example application MySchool.
  • Execute full junit test suite for OpenXava.
  • Help

Download base software needed

  • Download JDK from http://java.sun.com/j2se/1.5.0/download.html, and click Download JDK 5.0. Warning!, do not download J2EE or NetBeans.
  • Download JBoss 4.0.2 from www.jboss.org.
  • Download Eclipse 3.0.2 from www.eclipse.org
  • Download XmlBuddy (a XML plugin for Eclipse) from www.xmlbuddy.com (the free version).

Install and configure JDK

  • Install Java JDK.
  • Create a environment variable JAVA_HOME and set the value of JDK installation directory, for example JAVA_HOME=/opt/j2sdk1.5.0_03 or JAVA_HOME=C:\Program files\Java\j2sdk1.5.0_03.
  • Add java bin directory to your PATH, like this: export PATH=$JAVA_HOME/bin:$PATH (in UNIX case), in windows case PATH=%JAVA_HOME%\bin:%PATH% (in Control Pane > System)

Install and configure JBoss

  • Decompress jboss-4.0.2.zip in a directory to your choice (from now JBOSS_HOME).
  • Decompress openxava-db.zip inside JBOSS_HOME/server/default.
  • Execute JBOSS_HOME/bin/run.sh (in UNIX) or JBOSS_HOME\bin\run.bat (in Windows).
  • Wait to see in console 'JBoss (MX MicroKernel) ... Started in ...'
  • Step Ctrl-C and wait JBoss to stop.
  • Decompress again openxava-db.zip (a required step for a correct installation of databases)
  • Execute again run.sh or run.bat
  • When you see 'JBoss (MX MicroKernel) ... Started in ...' then Application Server is ready for server applications.

Install and configure Eclipse and OpenXava

  • Decompress eclipse-SDK-3.0.2.zip  in a directory to your choice (from now ECLIPSE_HOME).
  • Decompress XmlBuddy zip in ECLIPSE_HOME/plugins
  • Edit the file OPENXAVA_HOME/workspace/openxava.properties to set the correct path for your installation.
  • Execute ECLIPSE_HOME/eclipse
  • Eclipse prompts for workspace directory, write OPENXAVA_HOME/workspace. In case that you alredave Eclipse installed, you can change the worspace with File > Switch Workspace.
  • Within Eclipse go to Windows > Preferences > Java > Build Path > Classpath Variables and set the value of JBOSS_HOME variable.

Execute example application MySchool

The ant targets used are set in Eclipse, so to execute it only it is needed to click on arrow of External Tools button
  • Execute ant target MySchool.deployEJB.
  • Wait to see in JBoss console Started J2EE application: ... MySchool.ear
  • Execute ant target MySchool.desployWar.
  • Wait to that in console of JBoss appears [TomcatDeployer] deploy, ctxPath=/MySchool ...
  • Open a Internet Navigator and go to url http://localhost:8080/MySchool/xava/module.jsp?application=MySchool&module=Teachers
  • Now you can play with the application, and see how it works.

Execute junit test of MySchool application

  • Go to TeachersTest class (in src/org.openxava.school.tests) and with right button choose Run > JUnit Test
The OpenXava junit test allows to test a module like a real user with an Internet navigator. So it is possible to automatize the test application process. Your are encouraged to examine the TeachersTest code for make idea of how create a junit test for OpenXava. If you want to develop quality applications is essential to become fond to junit test.

Modify example application MySchool

Add other component and its module
Copy and paste components/Teacher.xml file with the new name Pupil.xml.
Change Pupil.xml until it remains:

<?xml version="1.0" encoding="ISO-8859-1"?>

<!DOCTYPE component SYSTEM "dtds/component.dtd">

<component name="Pupil">

    <entity>
        <property name="number" type="int" key="true"
            size="2" required="true"/>
        <property name="name" type="String"
            size="40" required="true"/>             
    </entity>
                       
    <entity-mapping table="MYSCHOOL@separator@PUPIL">
        <property-mapping property="number" column="NUMBER"/>
        <property-mapping property="name" column="NAME"/>
    </entity-mapping>
               
</component>

Elements in blue are the changed ones.
We have also to edit xava/application.xml, and add a new module:

<?xml version = "1.0" encoding = "ISO-8859-1"?>

<!DOCTYPE application SYSTEM "dtds/application.dtd">

<application name="MySchool">
   
    <module name="Teachers">
        <model name="Teacher"/>
        <controller name="Typical"/>
    </module>     
   
    <module name="Pupils">
        <model name="Pupil"/>
        <controller name="Typical"/>
    </module>     
                   
</application>

The blue things are the added ones.
Now only is needed to regenerate code and redeploy:
  • Execute target ant MySchool.generateEJB
  • Click on Build All
  • Execute target ant MySchool.deployEJB
  • Execute target ant MySchool.deployWar
  • Open a Internet navigator and go to url http://localhost:8080/MySchool/xava/module.jsp?application=MySchool&module=Pupils
  • And let's play...
That pupils know about his teachers
It is easy to make a pupil to have a reference to a teacher. Modify Pupil.xml :

<?xml version="1.0" encoding="ISO-8859-1"?>

<!DOCTYPE component SYSTEM "dtds/component.dtd">

<component name="Pupil">

    <entity>
        <property name="number" type="int" key="true"
            size="2" required="true"/>
        <property name="name" type="String"
            size="40" required="true"/>             
        <reference name="teacher"/>   
    </entity>
                       
    <entity-mapping table="MYSCHOOL@separator@PUPIL">
        <property-mapping property="number" column="NUMBER"/>
        <property-mapping property="name" column="NAME"/>
        <reference-mapping reference="teacher">
            <reference-mapping-detail column="TEACHER" referenced-model-property="id"/>
        </reference-mapping>
    </entity-mapping>
               
</component>

And now to test it:
  • Execute ant target MySchool.generateEJB
  • Click on Build All
  • Execute ant target MySchool.deployEJB
  • Execute ant target MySchool.deployWar
  • Open a Internet navigator an go to http://localhost:8080/MySchool/xava/module.jsp?application=MySchool&module=Pupils
As you see, plain vanilla.
If you want the reference to be displayed in combo format, you should change Pupil.xml :

<?xml version="1.0" encoding="ISO-8859-1"?>

<!DOCTYPE component SYSTEM "dtds/component.dtd">

<component name="Pupil">

    <entity>
        <property name="number" type="int" key="true"
            size="2" required="true"/>
        <property name="name" type="String"
            size="40" required="true"/>             
        <reference name="teacher"/>   
    </entity>
   
    <view>
        <reference-view reference="teacher">
            <descriptions-list/>
        </reference-view>
    </view>
                       
    <entity-mapping table="MYSCHOOL@separator@PUPIL">
        <property-mapping property="number" column="NUMBER"/>
        <property-mapping property="name" column="NAME"/>
        <reference-mapping reference="teacher">
            <reference-mapping-detail column="TEACHER" referenced-model-property="id"/>
        </reference-mapping>
    </entity-mapping>
               
</component>

And to deploy:
  • Click on Build All
  • Execute ant target MySchool.deployWar
  • Open a Internet navigator and go to url http://localhost:8080/MySchool/xava/module.jsp?application=MySchool&module=Pupils
And now let's go to put a pupils collection in teacher. Touch Teacher.xml and leave it this way:

<?xml version="1.0" encoding="ISO-8859-1"?>

<!DOCTYPE component SYSTEM "dtds/component.dtd">

<component name="Teacher">

    <entity>
        <property name="id" type="String" key="true"
            size="5" required="true"/>
        <property name="name" type="String"
            size="40" required="true"/>             
        <collection name="pupils">
            <reference model="Pupil"/>
        </collection>
    </entity>
                       
    <entity-mapping table="MYSCHOOL@separator@TEACHER">
        <property-mapping property="id" column="ID"/>
        <property-mapping property="name" column="NAME"/>
    </entity-mapping>
               
</component>

And deploy again:
  • Execute ant target MySchool.generateEJB
  • Click on Build All
  • Execute ant target MySchool.deployEJB
  • Execute ant target MySchool.deployWar
  • Open a Internet navigator and go to http://localhost:8080/MySchool/xava/module.jsp?application=MySchool&module=Teachers
That's all, folks
In this example you can see that little changes imply little effort and the effect is seen on instantaneous form.

Execute full junit test suite for OpenXava

The OpenXavaTest project has a double function, it is server as full junit test suite for OpenXava (currently more or less 270 tests), so it assures the constant robustness of OpenXava; and on the other hand is the official reference documentation of OpenXava. Everything possible, how and the behaviour of OpenXava is shown in OpenXavaTest.
Let's go:
  • Execute ant target OpenXavaTest.deployEJB
  • Execute ant target OpenXavaTest.deployWar
  • Execute XavaSuite. You can find it using  button.
