Welcome to this short voyage to start to use OpenXava.
These instructions suposses  that you have download, install and start openxava-1.2.
Here are the steps to begin to use OpenXava:
  • Execute example application MySchool.
  • Execute junit tests of MySchool application.
  • Modify example application MySchool.
  • Execute full junit test suite for OpenXava.
  • Help

Execute example application MySchool

OpenXava distribution comes with an example application called MySchool, in order to execute it, click the link 'MySchool' in left menu, or go to http://localhost:8080/openxava/portal/MySchool.
Now you can play with the application, and see how it works.

Execute junit test of MySchool application

Go to your Eclipse and in your MySchool project:
  • Go to TeachersTest class (in src/org.openxava.school.tests) and with right button choose Run > JUnit Test
The OpenXava junit test allows to test a module like a real user with an Internet navigator. So it is possible to automatize the test application process. Your are encouraged to examine the TeachersTest code for make idea of how create a junit test for OpenXava. If you want to develop quality applications is essential to become fond to junit test.

Modify example application MySchool

Add other component and its module
Copy and paste components/Teacher.xml file with the new name Pupil.xml.
Change Pupil.xml until it remains:

<?xml version="1.0" encoding="ISO-8859-1"?>

<!DOCTYPE component SYSTEM "dtds/component.dtd">

<component name="Pupil">

    <entity>
        <property name="number" type="int" key="true"
            size="2" required="true"/>
        <property name="name" type="String"
            size="40" required="true"/>             
    </entity>
                       
    <entity-mapping table="MYSCHOOL@separator@PUPIL">
        <property-mapping property="number" column="NUMBER"/>
        <property-mapping property="name" column="NAME"/>
    </entity-mapping>
               
</component>

Elements in blue are the changed ones.
We have also to edit xava/application.xml, and add a new module:

<?xml version = "1.0" encoding = "ISO-8859-1"?>

<!DOCTYPE application SYSTEM "dtds/application.dtd">

<application name="MySchool">
   
    <module name="Teachers">
        <model name="Teacher"/>
        <controller name="Typical"/>
    </module>     
   
    <module name="Pupils">
        <model name="Pupil"/>
        <controller name="Typical"/>
    </module>     
                   
</application>

The blue things are the added ones.
Now only that is needed to generate code and redeploy.
The ant targets used are set in Eclipse, so to execute it only it is needed to click on arrow of External Tools button
  • Execute target ant MySchool.generateEJB
  • Click on Build All
  • Execute target ant MySchool.deployEJB
  • Wait to see in JBoss console Started J2EE application: ... MySchool.ear
  • Execute target ant MySchool.deployPortlets
  • Wait to that in console of JBoss appears started Portlet Application at: MySchool
  • Go to http://localhost:8080/openxava/portal/MySchool/Pupils.psml
  • If portlet is not displayed, wait some seconds and click in the reload button of your browser.
  • And let's play...
That pupils know about his teachers
It is easy to make a pupil to have a reference to a teacher. Modify Pupil.xml :

<?xml version="1.0" encoding="ISO-8859-1"?>

<!DOCTYPE component SYSTEM "dtds/component.dtd">

<component name="Pupil">

    <entity>
        <property name="number" type="int" key="true"
            size="2" required="true"/>
        <property name="name" type="String"
            size="40" required="true"/>             
        <reference name="teacher"/>   
    </entity>
                       
    <entity-mapping table="MYSCHOOL@separator@PUPIL">
        <property-mapping property="number" column="NUMBER"/>
        <property-mapping property="name" column="NAME"/>
        <reference-mapping reference="teacher">
            <reference-mapping-detail column="TEACHER" referenced-model-property="id"/>
        </reference-mapping>
    </entity-mapping>
               
</component>

And now to test it:
  • Execute ant target MySchool.generateEJB
  • Click on Build All
  • Execute ant target MySchool.deployEJB
  • Execute ant target MySchool.deployPortlets
  • Go to http://localhost:8080/openxava/portal/MySchool/Pupils.psml
As you see, plain vanilla.
If you want the reference to be displayed in combo format, you should change Pupil.xml :

<?xml version="1.0" encoding="ISO-8859-1"?>

<!DOCTYPE component SYSTEM "dtds/component.dtd">

<component name="Pupil">

    <entity>
        <property name="number" type="int" key="true"
            size="2" required="true"/>
        <property name="name" type="String"
            size="40" required="true"/>             
        <reference name="teacher"/>   
    </entity>
   
    <view>
        <reference-view reference="teacher">
            <descriptions-list/>
        </reference-view>
    </view>
                       
    <entity-mapping table="MYSCHOOL@separator@PUPIL">
        <property-mapping property="number" column="NUMBER"/>
        <property-mapping property="name" column="NAME"/>
        <reference-mapping reference="teacher">
            <reference-mapping-detail column="TEACHER" referenced-model-property="id"/>
        </reference-mapping>
    </entity-mapping>
               
</component>

And to deploy:
  • Click on Build All
  • Execute ant target MySchool.deployPortlets
  • Go to http://localhost:8080/openxava/portal/MySchool/Pupils.psml
And now let's go to put a pupils collection in teacher. Touch Teacher.xml and leave it this way:

<?xml version="1.0" encoding="ISO-8859-1"?>

<!DOCTYPE component SYSTEM "dtds/component.dtd">

<component name="Teacher">

    <entity>
        <property name="id" type="String" key="true"
            size="5" required="true"/>
        <property name="name" type="String"
            size="40" required="true"/>             
        <collection name="pupils">
            <reference model="Pupil"/>
        </collection>
    </entity>
                       
    <entity-mapping table="MYSCHOOL@separator@TEACHER">
        <property-mapping property="id" column="ID"/>
        <property-mapping property="name" column="NAME"/>
    </entity-mapping>
               
</component>

And deploy again:
  • Execute ant target MySchool.generateEJB
  • Click on Build All
  • Execute ant target MySchool.deployEJB
  • Execute ant target MySchool.deployPortlets
  • Go to http://localhost:8080/openxava/portal/MySchool/Teachers.psml
That's all, folks
In this example you can see that little changes imply little effort and the effect is seen on instantaneous form.

Execute full junit test suite for OpenXava

The OpenXavaTest project has a double function, it is server as full junit test suite for OpenXava (currently more or less 290 tests), so it assures the constant robustness of OpenXava; and on the other hand is the official reference documentation of OpenXava. Everything possible, how and the behaviour of OpenXava is shown in OpenXavaTest.
Let's go:
  • Execute ant target OpenXavaTest.deployEJB
  • Execute ant target OpenXavaTest.deployWar
  • Execute XavaSuite. You can find it using  button.
