OpenXava es usado en producción con WebSphere Application Server y WebSphere Portal.

Cualquier war generado por OpenXava usando la tarea ant desplegarWar puede ser desplegado en un WebSphere Application Server, y cualquier war generado usando desplegarPortlets puede ser desplegado en un WebSphere Portal.

Comentarios:
  • No es aconsejable usar EJB2 (mejor usar Hibernate o EJB3 en su lugar), sin embargo OpenXava soporta EJB2 con CMP nativo para WebSphere, podemos verlo en las instrucciones WebSphere EJB2 o WebSphere Portal EJB2.
  • Si queremos usar EJB3 JPA necesitamos al menos un WebSphere 6.1.
  • WebSphere Application Server 5.0 está testeado con OpenXava hasta 1.2.1. No está certificado con OpenXava 2.x o superior.
  • WebSphere Portal 5.0 no está soportado.
  • Trabajar con WebSphere Application Server y WebSphere Portal requiere algo de experiencia.
