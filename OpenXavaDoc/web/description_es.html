OpenXava es un marco de trabajo para desarrollar fácilmente aplicaciones de gestión con XML y Java.

Su principal virtud reside en el hecho de que el corazón de nuestras aplicaciones es XML en vez de Java.
Por ejemplo, si quieres una aplicación J2EE para trabajar con Profesores solo necesitas escribir este sencillo archivo XML:

<?xml version="1.0" encoding="ISO-8859-1"?>

<!DOCTYPE componente SYSTEM "dtds/componente.dtd">

<componente nombre="Profesor">

  <entidad>
    <propiedad nombre="codigo" tipo="String"
      clave="true" longitud="5" requerido="true"/>
    <propiedad nombre="nombre" tipo="String"
      longitud="40" requerido="true"/>
  </entidad>

  <mapeo-entidad tabla="MYSCHOOL@separator@TEACHER">
    <mapeo-propiedad
      propiedad-modelo="codigo" columna-tabla="ID"/>
    <mapeo-propiedad
      propiedad-modelo="nombre" columna-tabla="NAME"/>
  </mapeo-entidad>

</componente>

Y nada más. Nada de clases Java, archivos XML para el mapeo, o dibujar páginas JSP o JSF, simplemente nada más.
OpenXava genera para tí una aplicación J2EE completa (con EJB CMP o Hibernate para la persistencia) como ésta, y con una interfaz de usuario para gestionar tus Profesores. La aplicación resultante te permite hacer manteniento, búsquedas, filtros, ordenaciones, generar informes en formato PDF, exporta a Excel, etc.
A pesar de su facilidad para desarrollo rápido, OpenXava es suficientemente flexible como para que puedas poner tu propio código en cualquier parte de la aplicación.
Con solo simples XMLs y código Java en formato JavaBean (clases de Java realmente reutilizables) puedes construir cualquier aplicación de gestión J2EE de una manera muy productiva. De hecho, muchas aplicaciones críticas de gestión, en producción actualmente, han sido desarrolladas con OpenXava.
