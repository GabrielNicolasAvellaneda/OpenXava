OpenXava Tutorial for core distribution

The core distribution has the complete OpenXava framework plus tests and examples. It's thought to be used by advanced OpenXava users in order to update their version of OpenXava to the last one, or simply that they prefer to configure its own portal or application server.
If you are new to OpenXava is far better to use the openxava-x.x.x-jetspeed2 distribution that comes with a Tomcat + Jetspeed + Eclipse Workspace all configured and ready to use, plus a local OpenXava Portal with documentation, reference guides, populated databases, etc.
Although if you know well Java, Ant and Tomcat you can follow this Tutorial and taste OpenXava.
These instructions suposses that you have:
  • Java JDK 5 or JDK 1.4.2.
  • Tomcat 5.5 or 5.0 installed and ready to work.
  • Ant 1.6.x.
Here are the steps to begin xaving:
  • Configure databases in Tomcat.
  • Execute example application MySchool.
  • Modify example application MySchool.
  • Help

Configure databases in Tomcat

The first step is creating and configure databases for running the demo applicacion:
  • Create a folder called data inside your Tomcat.
  • Copy from OpenXava/data the openxava-db.script and openxava-db.properties files to the data folder of Tomcat.
  • Copy from MySchool/data the my-school-db.script and my-school-db.properties files to the data folder of Tomcat.
Edit the conf/context.xml file of your Tomcat to declare the datasources, and add the next two:

<Context>
    ...
    <Resource name="jdbc/OpenXavaDS" auth="Container" type="javax.sql.DataSource"
       maxActive="20" maxIdle="5" maxWait="10000"
       username="sa" password="" driverClassName="org.hsqldb.jdbcDriver"
       url="jdbc:hsqldb:file:../data/openxava-db"/>

    <Resource name="jdbc/MySchoolDS" auth="Container" type="javax.sql.DataSource"
       maxActive="20" maxIdle="5" maxWait="10000"
       username="sa" password="" driverClassName="org.hsqldb.jdbcDriver"
       url="jdbc:hsqldb:file:../data/my-school-db"/>

</Context>

Now copy the OpenXavaTest/lib/hsqldb.jar file to common/lib folder of your Tomcat.
The database is ready to run.
Also you need:
  • Copy all files in lib folder from openxava distribution to common/lib folder of your Tomcat.
  • Edit the openxava.properties file (in root folder of openxava) and put in the tomcat.dir

Execute example application MySchool

OpenXava distribution comes with an example application called MySchool, in order to execute it follow the next steps, using Ant from command line:
  • cd MySchool
  • ant
  • Start your Tomcat.
  • Go to http://localhost:8080/MySchool/xava/module.jsp?application=MySchool&module=Teachers
  • And let's play...

Modify example application MySchool

Add other component and its module
Copy and paste components/Teacher.xml file with the new name Pupil.xml.
Change Pupil.xml until it remains:

<?xml version="1.0" encoding="ISO-8859-1"?>

<!DOCTYPE component SYSTEM "dtds/component.dtd">

<component name="Pupil">

    <entity>
        <property name="number" type="int" key="true"
            size="2" required="true"/>
        <property name="name" type="String"
            size="40" required="true"/>             
    </entity>
                       
    <entity-mapping table="MYSCHOOL@separator@PUPIL">
        <property-mapping property="number" column="NUMBER"/>
        <property-mapping property="name" column="NAME"/>
    </entity-mapping>
               
</component>

Elements in blue are the changed ones.
We have also to edit xava/application.xml, and add a new module:

<?xml version = "1.0" encoding = "ISO-8859-1"?>

<!DOCTYPE application SYSTEM "dtds/application.dtd">

<application name="MySchool">
   
    <module name="Teachers">
        <model name="Teacher"/>
        <controller name="Typical"/>
    </module>     
   
    <module name="Pupils">
        <model name="Pupil"/>
        <controller name="Typical"/>
    </module>     
                   
</application>

The blue things are the added ones.
Let's test our new module:
  • ant
  • Go to http://localhost:8080/MySchool/xava/module.jsp?application=MySchool&module=Pupils
  • And let's play...
That pupils know about his teachers
It is easy to make a pupil to have a reference to a teacher. Modify Pupil.xml :

<?xml version="1.0" encoding="ISO-8859-1"?>

<!DOCTYPE component SYSTEM "dtds/component.dtd">

<component name="Pupil">

    <entity>
        <property name="number" type="int" key="true"
            size="2" required="true"/>
        <property name="name" type="String"
            size="40" required="true"/>             
        <reference name="teacher"/>   
    </entity>
                       
    <entity-mapping table="MYSCHOOL@separator@PUPIL">
        <property-mapping property="number" column="NUMBER"/>
        <property-mapping property="name" column="NAME"/>
        <reference-mapping reference="teacher">
            <reference-mapping-detail column="TEACHER" referenced-model-property="id"/>
        </reference-mapping>
    </entity-mapping>
               
</component>

And now to test it:
  • ant
  • Go to http://localhost:8080/MySchool/xava/module.jsp?application=MySchool&module=Pupils
As you see, plain vanilla.
If you want the reference to be displayed in combo format, you should change Pupil.xml :

<?xml version="1.0" encoding="ISO-8859-1"?>

<!DOCTYPE component SYSTEM "dtds/component.dtd">

<component name="Pupil">

    <entity>
        <property name="number" type="int" key="true"
            size="2" required="true"/>
        <property name="name" type="String"
            size="40" required="true"/>             
        <reference name="teacher"/>   
    </entity>
   
    <view>
        <reference-view reference="teacher">
            <descriptions-list/>
        </reference-view>
    </view>
                       
    <entity-mapping table="MYSCHOOL@separator@PUPIL">
        <property-mapping property="number" column="NUMBER"/>
        <property-mapping property="name" column="NAME"/>
        <reference-mapping reference="teacher">
            <reference-mapping-detail column="TEACHER" referenced-model-property="id"/>
        </reference-mapping>
    </entity-mapping>
               
</component>

And to deploy:
  • ant
  • Go to http://localhost:8080/MySchool/xava/module.jsp?application=MySchool&module=Pupils
And now let's go to put a pupils collection in teacher. Touch Teacher.xml and leave it this way:

<?xml version="1.0" encoding="ISO-8859-1"?>

<!DOCTYPE component SYSTEM "dtds/component.dtd">

<component name="Teacher">

    <entity>
        <property name="id" type="String" key="true"
            size="5" required="true"/>
        <property name="name" type="String"
            size="40" required="true"/>             
        <collection name="pupils">
            <reference model="Pupil"/>
        </collection>
    </entity>
                       
    <entity-mapping table="MYSCHOOL@separator@TEACHER">
        <property-mapping property="id" column="ID"/>
        <property-mapping property="name" column="NAME"/>
    </entity-mapping>
               
</component>

And deploy again:
  • ant
  • Go to http://localhost:8080/MySchool/xava/module.jsp?application=MySchool&module=Teachers
That's all, folks
In this example you can see that little changes imply little effort and the effect is seen on instantaneous form.
