How can I insert our tailored functionality in a OpenXava component?

The blue pieces are objects that you can insert in our components to have our own functionality. We use the classic design pattern called Strategy (or Policy).

These are the Strategic Objects that we can insert:

  • Calculators (ICalculator): Used to implement business logic. Have been used to calculated properties, to calculate define methods.
  • Validators (IValidator): Used to validate the input data entered by users. We can define at property level or model level.
  • Converters (IConverter): Used to define conversion between Java types and database types. Needed to make the mapping. Very useful when you are using legacy databases.
  • Filters (IFilter): With our code allow to filter values to send to run one query with the database manager. It's required to develop real applications.

In adittion to the components there are other classes that allow to define how works a OpenXava application.

  • Actions (IAction): There are the objects that react to the requests of the user. Are defined in controllers.xml file..
  • Formatter (IFormatter): We can specify how to convert from an html form (String) to a Java type and vice versa. This way we can define the format of visualization of the information and his transformation to Java.
  • Generator (IPropertyCodeGenerator): We can define our own code generator for several properties.

All of these Strategy Objects can be configured giving values to the properties from XML. We can construct Strategy Objects configurables that we can reuse several times. Now we have more XML code and less Java code. We have sufficient functionality to adapt practically any need in a transactional application. But if you want you can write your completely EJB to manage the business logic, or the graphical interface using JSP. Though this characteristic exists to take advantage of already write code and is not necessary when one writes new code.
